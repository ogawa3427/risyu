この画面の見方
  • ページ読み込み直後は、まず古くてもいったんデータを表示し、その後に最新データへ差し替える挙動です。
  • uninitiated / initializing: 初回起動フェーズ。TSVが未準備なので待機するしかありません。
  • fresh: 十分新しいキャッシュを返している状態。現時点ではそのまま利用して問題ありません。
  • stale(first): バックエンドが次の更新準備を開始した直後。今表示中データが古いことは確定で、更新完了待ちです。
  • refreshing: 実スクレイピング実行中。完了後に lastCollectAt 変化を検知して表示を差し替えます。
